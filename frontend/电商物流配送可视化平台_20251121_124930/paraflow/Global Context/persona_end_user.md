# 终端用户画像 "安心期待者"

王晓雨

互联网公司产品经理,线上购物重度用户

**年龄:** 28岁

**地点:** 中国上海

> "我下单后最关心的就是包裹什么时候能到,能不能随时看到它现在在哪里,这样我就能安心工作而不用一直担心。"

## 关于王晓雨

王晓雨是一位工作繁忙的互联网产品经理,每周都会在各大电商平台购买服装、日用品和数码产品。她习惯在晚上或午休时间下单,但白天工作时经常会想起包裹的配送进度,担心错过签收时间或者包裹延误。她对物流服务的期望很高,希望能够像跟踪外卖一样实时看到包裹位置,并准确预估送达时间,这样她就可以安心工作,不用频繁查询或担心。

## 行为特征

- 手机是主要的购物和信息查询工具,习惯在碎片时间(通勤、午休、睡前)查看物流状态
- 下单后会在当天多次查询物流信息,如果看不到更新会感到焦虑
- 偏好直观的地图展示而非纯文字描述,希望"看见"包裹的实时位置
- 对送达时间预估的准确性非常在意,会据此安排自己的时间(如在家等待或委托他人代收)

## 痛点

- 传统物流查询只显示"已揽收、运输中、派送中"等模糊状态,无法知道包裹具体在哪里
- 物流信息更新不及时,有时候几个小时甚至大半天都看不到新的状态更新
- 预计送达时间不准确,经常显示"今日送达"但实际到晚上都没收到,打乱自己的计划
- 需要记住或复制订单号才能查询,操作繁琐,无法快速获取信息

## 目标

- 随时随地快速查询包裹的实时位置,不用担心物流信息滞后
- 通过地图可视化清晰看到包裹"在路上"的状态,获得心理安全感
- 获得准确的送达时间预估,合理安排自己的时间,避免错过签收
- 了解完整的物流进度历史,包括揽收、转运、派送等关键节点

## 核心任务

- 打开物流查询页面,输入订单号或从历史记录中选择订单
- 在地图上查看包裹的当前位置,观看实时移动动画,感知配送进度
- 查看预计送达时间,根据这个时间安排自己的行程或委托他人代收
- 浏览物流时间轴,了解包裹已经经过哪些节点,还有哪些环节待完成
- 在包裹到达后确认签收,结束本次物流追踪
